Current workflow

The plugin lives inside Unreal Editor. Use the built-in AI Workspace with your own Claude/OpenAI API key, or use UEHive as the Unreal-side MCP-style bridge for an external assistant workflow.

Install

Install UEHive from Fab

Use this path if you bought UEHive from Fab or added it to your Epic Games account. You do not need to copy plugin files manually.

01

Use the same Epic account

Sign in to Epic Games Launcher with the same account that purchased or claimed UEHive on Fab.

02

Refresh the Launcher

Fully close Epic Games Launcher, including the tray icon, then open it again. This often makes newly purchased plugins appear.

03

Install to Unreal Engine 5.7

In Unreal Engine > Library, find UEHive in your Fab or Vault library and install it to Unreal Engine 5.7.

04

Open your Unreal project

Start Unreal Engine 5.7 and open the project where you want to use UEHive. A clean test project is best for the first session.

05

Enable UEHive

If Unreal asks, go to Edit > Plugins, search for UEHive, enable it, and restart the editor once.

06

Open the AI Workspace

Open Window > UEHive > UEHive AI Workspace. If the entry is not visible, scroll the Window menu.

Important: API keys are optional depending on the workflow

UEHive does not include a shared vendor API key. The built-in AI Workspace needs your own Claude or OpenAI API key. The external MCP-style bridge workflow does not store an AI key inside UEHive, because the external assistant handles its own login, subscription, or provider access.

API or MCP

Choose how the AI connects to UEHive

UEHive v0.9.8 supports two practical workflows. Both use the same Unreal-side capabilities, but the AI access is handled differently.

Built-in AI Workspace

Use this when you want to chat with Claude or OpenAI directly inside Unreal. This mode requires your own Claude or OpenAI API key, stored locally under Saved/UEHive for that Unreal project.

MCP-style external bridge

Use this when you prefer an external assistant workflow. UEHive stays open in Unreal and exposes the Unreal-side bridge, project context, and supported Blueprint/UMG capabilities to the external assistant.

What "no API key" means

It means no Claude/OpenAI key is stored inside UEHive. It does not mean the AI model access is included. The external app still handles its own account, login, subscription, or API access.

How to use the bridge

Open your Unreal project, enable UEHive, and keep the editor running. UEHive starts its local bridge at http://127.0.0.1:17857/uehive/swarm when the plugin is active.

Confirm the bridge is online

In UEHive, the System panel should show Runtime ready or Workspace ready. Advanced users can also check the local health route through their configured connector: /uehive/swarm/health.

External assistant setup

Open the external assistant from the Unreal project root when possible. Use Copy for Codex when you want a clean handoff prompt that includes project context and UEHive capabilities.

Use UEHive as an external assistant bridge

  1. Open the Unreal project that contains UEHive and keep Unreal Editor running.
  2. Enable UEHive in Edit > Plugins if it is not already enabled, then restart Unreal once.
  3. Open Window > UEHive > UEHive AI Workspace so the runtime and bridge status are visible.
  4. Confirm the System panel shows Runtime ready or Workspace ready.
  5. Open your external assistant from the Unreal project root folder when possible.
  6. Use the external assistant's MCP or tool setup to reach UEHive's local bridge endpoint: http://127.0.0.1:17857/uehive/swarm.
  7. If your project has bridge authentication enabled, use the project-local UEHive bridge token or secret only on your own machine. Do not share it publicly.
  8. Start with a safe test request, such as inspecting an open Blueprint or creating a Blueprint in a selected test folder.

Quick decision

  1. Want everything inside Unreal? Use the AI Workspace and add your own Claude/OpenAI API key.
  2. Do not want to paste an API key into UEHive? Use the MCP-style external bridge with your external assistant.
  3. Need the safest first test? Use the AI Workspace on /Game/UEHiveWorkspaceTests or a selected test folder.

AI Workspace

Use Claude or OpenAI directly inside UEHive

The integrated workspace keeps the main workflow inside Unreal: prompt, context, AI response, preview plan, approval, and native execution.

1. Open UEHive AI Workspace

In Unreal Editor, use Window > UEHive > UEHive AI Workspace.

2. Choose Claude or OpenAI

Select the provider, choose a recommended model from the model menu, and paste your own API key. Keys are stored locally under Saved/UEHive.

3. Write the request

Describe what you want in natural language. UEHive resolves the target from the selected or open Blueprint for inspect/scaffold tasks, and from the active Content Browser folder for creation tasks.

4. Preview before execution

UEHive shows the AI action plan first. Nothing modifies assets until you click Execute Plan.

5. Execute approved plans

Approved plans reuse UEHive's native capabilities for Blueprint actors, Widget Blueprints, widget layouts, graph scaffolds, inspections, and level placement.

6. Keep Codex optional

The Copy for Codex button creates a clean prompt if you want to hand off a larger coding task to Codex outside Unreal.

Troubleshooting

Common buyer issues

These fixes cover the most common first-day problems: the plugin not appearing, the panel not opening, or the AI tool looking at the wrong folder.

I purchased it but cannot download it

Fully quit Epic Games Launcher, reopen it, and check your Fab or Vault library again. If it still does not appear, sign out and sign back in with the purchase account.

The Launcher still does not show UEHive

Restart Windows and open the Launcher again. If the library is still stale, reinstalling Epic Games Launcher can force a clean refresh. You normally do not need to reinstall Unreal Engine.

UEHive is installed but disabled

Open Edit > Plugins, search for UEHive, enable it, then restart Unreal Editor when prompted.

I cannot find the panel

Open Unreal Editor's Window menu and scroll through the list. If the plugin was enabled during the same session, restart the editor once.

UEHive cannot resolve a target

For inspections or graph scaffolds, open or select a Blueprint first. For new assets, select the destination folder in the Content Browser before sending the request.

I do not have an API key

You can still use UEHive as the Unreal-side bridge with an external assistant workflow. UEHive will not store an AI key in that mode, but the external assistant still needs its own account, subscription, or provider access.

The external assistant cannot connect

Keep Unreal Editor open, confirm the UEHive System panel shows ready, and make sure the external tool is using the active project root and local bridge endpoint. If bridge auth is enabled, verify the local token or secret.

Support checklist

When asking for help, include your Unreal version, Windows version, AI tool name, whether UEHive appears in Edit > Plugins, and any error text.

Release Notes

What changed in v0.9.8

This update focuses on making UEHive feel more like an in-editor assistant while keeping execution controlled and reviewable.

Assistant-style workspace

A cleaner chat-first UI lets buyers discuss the task, review the generated plan, and execute only after approval.

Smarter target handling

UEHive now uses the open or selected Blueprint for inspection/scaffold tasks and the active Content Browser folder for asset creation.

More gameplay scaffolds

Validated examples include sliding doors, trigger lights, rotating pickups, simple patrol actors, and procedural scatter systems.

Project memory

Successful workflows are saved as reusable local skills so future sessions can reuse proven patterns instead of starting from scratch.

API or external bridge choice

Buyers can use the built-in workspace with their own Claude/OpenAI API key, or keep AI authentication outside UEHive through the MCP-style external bridge workflow.

Clearer safety defaults

Plans stay in preview mode until the user approves execution, and UEHive avoids editing Engine content.
