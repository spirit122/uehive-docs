Install

Install UEHive in minutes

The normal buyer workflow is simple: install UEHive from Epic Games Launcher, open Unreal Engine, and launch the panel. No manual file copy is required.

01

Add UEHive to your Fab Library

Purchase or claim UEHive on Fab so it appears in your library.

02

Install it from Epic Games Launcher

Open Unreal Engine > Library, find UEHive in the Fab Library, and install it to Unreal Engine 5.7.

03

Open your project in Unreal Engine

Start Unreal Engine 5.7 and open an existing project or a clean test project.

04

Enable the plugin if Unreal asks for it

If UEHive is disabled, go to Edit > Plugins, search for UEHive, enable it, and restart the editor once.

05

Open the UEHive panel

Open the Window menu in Unreal Editor, scroll until you see the UEHive section, and click Native Fabric.

06

Pick the easiest startup mode

For the fastest local Codex-ready setup, choose Internal Swarm. Use Fab Safe later when preparing a packaged demo or marketplace build.

Where do I find the panel in Unreal?

In Unreal Editor, open the Window menu and look for the UEHive section in the list. The panel entry appears there as Native Fabric. If you do not see it immediately, scroll down in the Window menu until the UEHive section becomes visible.

Simplest Codex Connection Path

Install UEHive once from the Launcher, open the Window menu, click Native Fabric under the UEHive section, keep the project open, and use Internal Swarm for the shortest local Codex workflow. UEHive exposes its local bridge from the editor session, so buyers do not need to copy the plugin manually or search for it from outside Unreal just to get started.

Where should I open Codex?

Install UEHive to Unreal Engine, but open Codex on your Unreal project. After the plugin is installed and enabled, start Codex from the project root folder that contains the .uproject, Content, Config, and Saved folders. Do not target the Engine installation folder unless you are developing UEHive itself.

First Use

How to start using UEHive after installation

These steps help new users get value quickly without needing to guess the workflow.

1. Choose the right first-run mode

The shortest path for a local Codex session is Internal Swarm. Use Fab Safe when you want safer packaged demos, and Local Solo Mapping when you want a lower-risk local pass first.

2. Start with the included showcase

Open the built-in showcase map under /UEHive/FabShowcase/Maps/L_UEHive_FabBlueprintShowcase to inspect the shipped examples immediately.

3. Confirm the bridge is online

Open the UEHive panel and confirm the local bridge is online before trying deeper Codex-driven workflows. This is the quickest sanity check that the editor session is ready.

4. Open Codex on the project root

Launch Codex from the Unreal project folder, not from the Engine install directory. This gives Codex the correct project context, content paths, config files, and UEHive skill memory.

5. Run your first authoring workflow

The current release is built around practical editor workflows such as Blueprint actor creation, Widget Blueprint generation, UI layout scaffolds, and targeted gameplay interaction recipes.

6. Keep project memory between sessions

UEHive persists reusable project memory under Saved/Skills, so later sessions can reuse what was already mapped or observed.

7. Use a test map before touching production content

The best onboarding path is to start in a clean test level, validate your first Blueprint workflows, and then move into your main project areas.

Recommended first session

  1. Install UEHive from the Fab Library in Epic Games Launcher.
  2. Open Unreal Engine 5.7, open the Window menu, and click Native Fabric in the UEHive section.
  3. Select Internal Swarm if you want the shortest local Codex-ready setup.
  4. Open Codex on the project root folder, not on the Engine installation folder.
  5. Open the built-in showcase map and inspect the included Blueprint examples.
  6. Create your first test Blueprint in a non-production map.
