Current workflow

The plugin lives inside Unreal Editor. The AI conversation still happens in your chosen AI tool, while UEHive provides the Unreal project bridge.

Install

Install UEHive from Fab

Use this path if you bought UEHive from Fab or added it to your Epic Games account. You do not need to copy plugin files manually.

01

Use the same Epic account

Sign in to Epic Games Launcher with the same account that purchased or claimed UEHive on Fab.

02

Refresh the Launcher

Fully close Epic Games Launcher, including the tray icon, then open it again. This often makes newly purchased plugins appear.

03

Install to Unreal Engine 5.7

In Unreal Engine > Library, find UEHive in your Fab or Vault library and install it to Unreal Engine 5.7.

04

Open your Unreal project

Start Unreal Engine 5.7 and open the project where you want to use UEHive. A clean test project is best for the first session.

05

Enable UEHive

If Unreal asks, go to Edit > Plugins, search for UEHive, enable it, and restart the editor once.

06

Open the panel

Open the Window menu in Unreal Editor and look for the UEHive panel. If the entry is not visible, scroll the Window menu.

Important: where to open your AI tool

Open Codex or your other AI coding tool on the Unreal project root folder, the folder that contains the .uproject, Content, Config, and Saved folders. Do not open it on the Unreal Engine installation folder unless you are developing engine code.

Connect AI

How UEHive works with Claude, Codex, ChatGPT, or Gemini

UEHive is not a replacement chat app. It is the native Unreal bridge that gives your AI workflow better project context and editor-side execution paths.

1. Keep Unreal Editor open

UEHive works from the active editor session, so open your Unreal project and keep it running.

2. Open your AI app normally

Use Claude, Codex, ChatGPT, Gemini, or your preferred MCP-capable workflow for the conversation.

3. Point the AI tool at your project

If the tool asks for a folder, choose the Unreal project folder that contains the .uproject file.

4. Use UEHive for Unreal-side work

Ask your AI tool to work through UEHive when the task needs Unreal project knowledge, Blueprint authoring support, or editor-side validation.

5. Start with safe test content

Use a test map or test Blueprint first. Confirm the generated result before applying the workflow to production content.

6. Reuse project memory

UEHive keeps reusable project knowledge under Saved/Skills so later sessions can build on previous observations.

Troubleshooting

Common buyer issues

These fixes cover the most common first-day problems: the plugin not appearing, the panel not opening, or the AI tool looking at the wrong folder.

I purchased it but cannot download it

Fully quit Epic Games Launcher, reopen it, and check your Fab or Vault library again. If it still does not appear, sign out and sign back in with the purchase account.

The Launcher still does not show UEHive

Restart Windows and open the Launcher again. If the library is still stale, reinstalling Epic Games Launcher can force a clean refresh. You normally do not need to reinstall Unreal Engine.

UEHive is installed but disabled

Open Edit > Plugins, search for UEHive, enable it, then restart Unreal Editor when prompted.

I cannot find the panel

Open Unreal Editor's Window menu and scroll through the list. If the plugin was enabled during the same session, restart the editor once.

My AI tool cannot see the project

Close the AI workspace and reopen it on the Unreal project root folder, not on UE_5.7, Engine, or the plugin install folder.

Support checklist

When asking for help, include your Unreal version, Windows version, AI tool name, whether UEHive appears in Edit > Plugins, and any error text.
